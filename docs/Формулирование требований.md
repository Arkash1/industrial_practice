
# Формулирование требований

Разрабатываемый программный продукт должен обеспечивать выполнение следующих **функциональных требований**:

- Возможность **генерации QR-кодов** для различных типов данных (текст, URL, Wi-Fi, контакт, email, телефон, SMS, геолокация)
- Возможность **Кастомизации внешнего вида** QR-кода (выбор цвета переднего плана и фона)
- Возможность **Сканирования QR-кодов** с использованием камеры устройства в реальном времени
- **Автоматическое сохранение** созданных и отсканированных кодов в локальную историю
- **Просмотр истории** операций с возможностью фильтрации (все, созданные, отсканированные) и поиска
- Возможность **добавления кодов в "Избранное"** для быстрого доступа
- **Сохранение изображения QR-кода** в галерею устройства в формате PNG
- Наличие функции **"Поделиться"** изображением или содержимым кода через сторонние приложения
- Возможность **Удаления записей** из истории (по одной или полная очистка)

К программному продукту предъявляются следующие **нефункциональные требования**:

- **Удобный и современный пользовательский интерфейс**, соответствующий гайдлайнам Material Design 3
- **Высокая производительность** (время генерации кода не более 500 мс, быстрый запуск приложения)
- **Стабильная работа приложения** без аварийных завершений
- **Надежное локальное хранение данных** с использованием базы данных SQLite (Room)
- **Обеспечение конфиденциальности** (отсутствие передачи пользовательских данных на внешние серверы)
- **Работа приложения в автономном режиме** без необходимости подключения к сети Интернет
- **Корректная обработка разрешений** (на камеру и запись в память) с понятными пользователю пояснениями
- **Поддержка темной и светлой тем** оформления интерфейса
